1. 문제 상황

QA 자동화 툴을 만들면서 가장 큰 난관은 "사용자의 클릭을 어떻게 코드로 바꿀 것인가?" 였습니다.
단순히 event.target을 잡으면 <div> 안에 있는 <span>이 잡히거나, 동적으로 생성된 클래스명을 가진 요소가 잡혀서 재사용 불가능한 코드가 생성되었습니다.

2. 해결 방법: XPath 생성 알고리즘

이를 해결하기 위해 우선순위 기반 XPath 생성기를 만들었습니다.

ID가 있는가? -> //\*[@id="login-btn"] (가장 명확)

data-testid가 있는가? -> 개발자가 심어둔 테스트용 ID 활용

Unique ClassName인가? -> 클래스명 활용

최악의 경우: 부모 요소를 거슬러 올라가며 div[2]/span[1] 형태로 구조적 경로 생성

3. 결론

이 알고리즘을 통해 복잡한 React 앱에서도 99% 정확도로 Selenium 코드를 역변환하는 데 성공했습니다.
